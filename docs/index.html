为什么选择自建

数据保存在本地

节点和订阅信息只写入你自己的 ./data 目录，不上传分析数据，也没有埋点。

一批节点，多种输出

同一批节点可以生成 Raw / Base64 / Surge / Clash 四种订阅格式，分别给不同客户端使用，不需要维护多份节点列表。

迁移只带数据目录

带走 ./data 和 .env，在新机器上执行 docker compose up -d 就能恢复，不需要再导入一遍。

快速部署

1. 复制配置文件

git clone https://github.com/baixiaoshengofficial/SubscribeManager.git
cd SubscribeManager
cp .env.example .env
# 编辑 .env：账号密码、端口、SESSION_SECRET 等按需修改

2. 启动服务

docker compose up -d
# 或源码部署：
make install && make frontend-build && make backend-dev

3. 打开它

http://<你的主机>:<FRONTEND_PORT>/
# 默认账号 admin / admin（请在 .env 改掉）
